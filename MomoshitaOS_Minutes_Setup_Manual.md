# 「ももしたOS用議事録」自動生成システムの導入マニュアル

Google Meetで自動作成された会議メモ（議事録）から、自動で指定テンプレートに成形した「ももしたOS用議事録」を作成し、元のメールスレッドに返信するシステムを構築しました。

作成したスクリプトファイル：
- [meeting_formatter.js](file:///Users/moriokarin/.gemini/antigravity/brain/abc8f0e4-d602-472d-a611-152aa8feb4f6/scratch/meeting_formatter.js)

以下に、りんさんがこのシステムを導入・設定するための手順をわかりやすく解説します。

---

## 導入手順

### Step 1. Gemini API キーの取得
GASからGeminiを呼び出すために、APIキーを無料で取得します。

1. [Google AI Studio](https://aistudio.google.com/) にアクセスします。
2. Googleアカウントでサインインします。
3. 画面左上にある **「Get API key」** ボタンをクリックします。
4. **「Create API key」** をクリックし、任意のプロジェクトを選択（または新規作成）してAPIキーを生成します。
5. 生成されたAPIキー（`AIzaSy...` で始まる文字列）をコピーし、手元に控えておきます。

---

### Step 2. Google Apps Script (GAS) プロジェクトの作成
Googleのクラウド上でプログラムを実行するための環境を作成します。

1. [Google Apps Script](https://script.google.com/) にアクセスします。
2. **「新しいプロジェクト」** ボタンをクリックします。
3. プロジェクト名（画面左上の「無題のプロジェクト」）をクリックし、分かりやすい名前（例: `MeetMinutesFormatter`）に変更します。
4. 元々エディタにある `function myFunction() {}` をすべて削除します。
5. [meeting_formatter.js](file:///Users/moriokarin/.gemini/antigravity/brain/abc8f0e4-d602-472d-a611-152aa8feb4f6/scratch/meeting_formatter.js) の内容をコピーして、エディタに貼り付けます。
6. フロッピーディスクのアイコン（または `Ctrl + S` / `Cmd + S`）を押して保存します。

---

### Step 3. Gemini API キーの設定
安全にAPIキーをプログラムに読み込ませるための設定を行います。

1. GASエディタの左メニューにある **「プロジェクトの設定（歯車マーク）」** をクリックします。
2. 画面を下部にスクロールし、**「スクリプトプロパティ」** セクションを探します。
3. **「スクリプトプロパティを追加」** をクリックします。
4. 以下のように入力します：
   - **プロパティ**: `GEMINI_API_KEY`
   - **値**: Step 1 でコピーした Gemini API キー
5. **「スクリプトプロパティを保存」** ボタンをクリックします。

---

### Step 4. 初回実行と権限の承認（テスト）
プログラムが正常に動作するかテストし、Googleアカウントへのアクセス権限を許可します。

> [!NOTE]
> 初回テストを行う前に、ご自身のGmailに「メモ: 「(予定の名前)」（日付）」という形式の件名で届いているメールが1通以上あることを確認してください。（ない場合はテスト用に自分宛てにダミーのメールを作成するか、実際のメールが届くまでお待ちください）

1. エディタ画面（左メニューのコードマーク `<>`）に戻ります。
2. 画面上部の実行する関数が `processMeetMinutes` になっていることを確認し、**「実行」** ボタンをクリックします。
3. **「承認が必要です」** というポップアップが表示されるので、**「権限を確認」** をクリックします。
4. ご自身のアカウントを選択します。
5. 「このアプリは Google で確認されていません」と警告が出た場合は、左下の **「詳細」** をクリックし、**「MeetMinutesFormatter（安全ではないページ）に移動」** をクリックします。
6. アクセスを許可します（Gmailの読み書き、ドキュメントの読み取りに必要な権限です）。
7. 実行が開始され、ログが表示されます。「すべての処理が完了しました。」と表示されれば成功です。対象のメールスレッドにテンプレート通りの返信が自動で届いているかご確認ください。

---

### Step 5. 自動実行（トリガー）の設定
会議が終わった後、放置していても自動で実行されるようにスケジュールを設定します。

1. GASエディタの左メニューにある **「トリガー（目覚まし時計マーク）」** をクリックします。
2. 画面右下の **「トリガーを追加」** ボタンをクリックします。
3. 以下のように設定します：
   - **実行する関数を選択**: `processMeetMinutes`
   - **実行するデプロイを選択**: `Head`
   - **イベントのソースを選択**: `時間主導型`
   - **時間ベースのトリガーのタイプを選択**: `分ベースのタイマー`
   - **時間の間隔を選択（分）**: `5分ごと`（またはお好みの間隔）
4. **「保存」** をクリックします。

これで、5分ごとにGmailをチェックし、新しい議事録メール（件名が「メモ: 「...」」）が届くと自動的に「ももしたOS用議事録」が作成されて返信されるようになります！

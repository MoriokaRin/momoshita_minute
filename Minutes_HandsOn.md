## 1. はじめに ― この資料でできるようになること

桃下村塾の会議で、こんな経験はありませんか？

- 「誰が議事録を書くの？」で毎回もめる
- 議事録を書いている間、議論に集中できない
- あとから「あの会議で何が決まったっけ？」と思い出せない

**Google Meetの「文字起こし」と「自動メモ生成（Gemini）」機能**を使えば、これらの悩みがすべて解決します。

会議中にAIが自動で文字起こしをして、会議が終わると**要約された議事録がメールで届く**ようになります。しかも、この方法は**対面の会議でもスマホやPCを1台置くだけ**で使えるのがポイントです。

## 2. 準備するもの（前提条件）

この資料の手順を実行するために必要なものは以下の通りです。

| 必要なもの | 説明 |
|---|---|
| **Googleアカウント** | Gmailアドレスがあれば大丈夫です |
| **Google Workspace（有料プラン）** | 文字起こし・自動メモ生成機能を使うには、**会議の主催者（ホスト）**が「Business Standard」以上のプラン、または「Google AI Pro / Ultra」に加入している必要があります。※参加者は無料アカウントでもOKです |
| **パソコン（またはスマートフォン）** | Google Meetに接続できるデバイス |
| **インターネット環境** | Wi-Fiやモバイル通信など |
| **マイク** | 対面会議の場合、会議室の音声を拾うために必要です。PC内蔵マイクでも可ですが、集音マイクがあるとより精度が上がります |

> **💡 ポイント**
> 文字起こし機能が使えるかどうかは、**会議の「主催者」のプラン**で決まります。参加者個人のプランは関係ありません。桃下村塾のアカウントが対応プランであればOKです。

## 3. 基本編：文字起こしありのMeet会議のはじめ方

### 3-1. オンライン会議の場合

通常のオンライン会議で文字起こしと自動メモ生成を有効にする方法です。

#### 方法A：会議中に手動で開始する

1. Google Meetの会議に参加します。
2. 画面右上（またはメニュー内）にある**鉛筆アイコン（「メモを取る」または「自動メモ生成」）**をクリックします。

<!-- 📸 ここにスクリーンショットを挿入：Meetの画面で「メモを取る」アイコンの位置を示す画像 -->

3. メモ生成のオプションが表示されるので、**「開始」**をクリックします。
4. 画議参加者全員に「メモが記録されています」という通知が表示されます。
5. 会議終了後、自動的にGoogleドキュメントに要約が保存され、主催者にメールで通知が届きます。

#### 方法B：事前にGoogleカレンダーから自動設定する（おすすめ！）

毎回ボタンを押す手間を省きたい場合は、**会議を作成するときに事前設定**しておくのがおすすめです。

1. **Googleカレンダー**（https://calendar.google.com/）を開きます。
2. 新しい予定を作成し、「Google Meetのビデオ会議を追加」をクリックします。

<!-- 📸 ここにスクリーンショットを挿入：カレンダーでMeet会議を追加する画面 -->

3. 会議の設定（歯車アイコン）を開きます。
4. **「自動メモ生成」をオンに**します。

<!-- 📸 ここにスクリーンショットを挿入：自動メモ生成をオンにする設定画面 -->

5. 予定を保存します。
6. これで、この会議が始まると**自動的に文字起こしとメモ生成が開始**されます。

> **💡 ポイント**
> Google Meetの設定画面から、自分が主催する**すべての会議**で自動メモ生成をデフォルトでオンにすることも可能です。頻繁に会議がある方におすすめです。

---

### 3-2. 対面（オフライン）会議の場合

**「対面の会議なのにGoogle Meetを使うの？」**と思うかもしれませんが、ここがこのテクニックの最大のポイントです。

会議室にパソコンを1台置いて、Google Meetを「録音マイク代わり」に使うことで、対面の会話も自動で文字起こし・議事録化できます。

#### 手順

1. 会議室の**テーブル中央付近**にパソコンを1台設置します。
   - 参加者全員の声がマイクに届く位置に置きましょう。
   - 外付けの集音マイク（スピーカーフォンなど）があると、さらに精度が上がります。

2. そのパソコンでGoogle Meetの会議を**1人で開始**します。
   - URLは https://meet.google.com/ からアクセスし、「新しい会議を作成」→「会議を今すぐ開始」をクリックします。

<!-- 📸 ここにスクリーンショットを挿入：Google Meetのホーム画面 -->

3. 「1人です」という表示が出ることがありますが、そのまま続行してOKです。

4. 文字起こし・自動メモ生成を**開始**します（前述の「方法A」と同じ手順です）。

5. パソコンはそのまま置いておき、**普通に対面で会議を進めます。**
   - Meetの画面は最小化しておいても大丈夫です。

6. 会議が終わったら、Meetの通話を終了します。
   - しばらくすると、自動で議事録メールが届きます。

> **⚠️ 注意：ハウリング（音のループ）を防ぐために**
> もし他の参加者も自分のPCでMeetを開いている場合（リモート参加者がいる場合など）は、**会議室内にいる人のPCは全員マイクをミュートにする**か、**「コンパニオンモード」**で入室してください。コンパニオンモードなら、マイク・カメラが自動でオフになるため、音のループを防げます。

> **💡 話者識別のコツ**
> AIは「誰が話したか」を完全に判別するのが難しい場合があります。発言の冒頭に**「〇〇です。」と名前を名乗ってから話す**ルールを作ると、後から議事録を見返す際にとても便利です。

## 4. 💡 知っておきたい設定：対面利用時の「自動終了」を防ぐ方法

対面会議でMeetを使う場合、**初期設定のままだと「参加者がいない通話」として自動的に終了してしまう**ことがあります。

せっかく文字起こしを開始したのに、途中でMeetが勝手に終了してしまったら大変です。以下の設定で防ぎましょう。

> **📌 出典**
> この設定は、[原田悠太さん(@yuta_harada_)のXの投稿](https://x.com/yuta_harada_/status/2069781457619542284) で紹介されているテクニックです。

### 設定手順

1. **Google Meetのホーム画面**（https://meet.google.com/）にアクセスします。
2. 画面右上にある **⚙️ 歯車マーク（設定）** をクリックします。

<!-- 📸 ここにスクリーンショットを挿入：Meetホーム画面の設定アイコンの位置 -->

3. 設定メニューの中から **「参加者がいない通話を終了」** という項目を探します。
4. この項目を **「オフ」** にします。

<!-- 📸 ここにスクリーンショットを挿入：「参加者がいない通話を終了」をオフにする画面 -->

5. 設定を閉じます（自動保存されます）。

これで、1人だけの通話でも途中で勝手に終了されることがなくなります！

## 5. 会議が終わったら：自動で届く議事録メールとGoogleドキュメント

### 5-1. どんなメールが届くの？

会議が終了すると、しばらくして（通常数分〜十数分後）**Gmailに自動で議事録メールが届きます。**

- **送信元**: `gemini-notes@google.com`（Google公式の自動送信アドレス）
- **件名**: 「メモ: 「(予定の名前)」（日付）」という形式の件名
- **受信者**: 会議の主催者、およびメモ生成を開始した参加者

メールの内容には以下のような情報が含まれます：

- **会議の要約**（AIが自動で作成した議論のポイント）
- **決定事項**
- **次のステップ（アクションアイテム）**
- **Googleドキュメントへのリンク**（より詳細な文字起こしや議事録全文）

<!-- 📸 ここにスクリーンショットを挿入：実際に届いた議事録メールのサンプル -->

### 5-2. Googleドキュメント（議事録ファイル）の保存場所

メールに記載されているリンクをクリックすると、Googleドキュメント形式の詳細な議事録が開きます。
このファイルは**自動的にGoogleドライブに保存**されています。

#### 保存先ディレクトリ（フォルダ）

```
マイドライブ > Meet Recordings
```

- **保存先のドライブ**: 会議の**主催者（ホスト）のGoogleドライブ**に保存されます。自分が主催者でない場合は、自分のドライブには保存されません（メールのリンクからアクセスできます）。

<!-- 📸 ここにスクリーンショットを挿入：Googleドライブの「Meet Recordings」フォルダの場所 -->

#### 保存されるファイルの種類

| ファイルの種類 | 形式 | 内容 |
|---|---|---|
| 文字起こし | Googleドキュメント (.gdoc) | 話者名・タイムスタンプ付きの会話テキスト |
| 会議メモ（要約） | Googleドキュメント (.gdoc) | AIが作成した要約・決定事項・アクションアイテム |
| 録画データ（※録画した場合のみ） | MP4ファイル | 会議の動画 |

> **💡 ファイルが見つからない場合**
> 「マイドライブ」に `Meet Recordings` フォルダが見当たらない場合は、別のGoogleアカウント（特に会議を主催したアカウント）でGoogleドライブにログインして確認してみてください。
> また、Googleカレンダーの該当する予定を開くと、関連するドキュメントへのリンクが自動的に添付されていることが多いです。

## 6. 発展編①：GASとGemini APIで「ももしたOS用議事録」を自動生成する

ここからは発展的な内容です。Google Meetが自動で作成した議事録を、**桃下村塾の独自テンプレート（「ももしたOS用議事録」）に自動で整形して、元のメールスレッドに返信**するシステムを構築します。

### 6-1. このシステムでできること

1. Google Meetの会議が終わると、Googleから議事録メールが届く（ここまでは基本編の範囲）
2. **Google Apps Script（GAS）**というGoogleの無料プログラミング環境が、定期的にGmailをチェック
3. 新しい議事録メールを見つけると、**Gemini AI**に内容を渡して、以下のテンプレートに自動成形
4. 成形された「ももしたOS用議事録」が、**元のメールスレッドへの返信**として自動送信される

#### ももしたOS用議事録テンプレート

```
議事録

📅 開催日: yyyy/mm/dd
🕐 時間:
📍 場所:
👥 参加者:

────────────────

■ アジェンダ

1.
2.
3.

────────────────

■ 会議中メモ
（議論しながら気づいたこと、参考リンク、雑談で出た発言など、自由に書いてOK）

────────────────

■ 議論内容

【議題1】
・
・

【議題2】
・

────────────────

■ 決定事項

・
・

────────────────

■ アクションアイテム

担当:
やること:
期限:

────────────────

■ 次回予定

・
```

### 6-2. 必要なもの

| 必要なもの | 説明 | 費用 |
|---|---|---|
| Googleアカウント | GASを利用するために必要 | 無料 |
| Gemini APIキー | AIに議事録の成形を指示するために必要 | 無料枠あり（通常利用では十分） |

> **💡 パソコンの負荷について**
> このシステムは、ご自身のパソコンではなく**Googleのサーバー（クラウド）上で動作**します。パソコンの電源が切れていても、ブラウザを閉じていても、自動的にGoogleが裏側で実行してくれます。パソコンのメモリやバッテリーを消費することはありません。

### 6-3. Step 1: Gemini APIキーの取得

GASからGemini（AI）を呼び出すために、「APIキー」と呼ばれるパスワードのようなものを取得します。

1. **Google AI Studio**（https://aistudio.google.com/）にアクセスします。
2. Googleアカウントでサインインします。
3. 画面左メニューの最上部にある **「Get API key」**（鍵マークのアイコン）をクリックします。

<!-- 📸 ここにスクリーンショットを挿入：Google AI Studioの画面で「Get API key」の位置 -->

4. 画面中央に表示される **「Create API key」** という青いボタンをクリックします。

<!-- 📸 ここにスクリーンショットを挿入：「Create API key」ボタン -->

5. ポップアップ画面が表示されます。

   - **「キー名（API Key Name）」** を入力する欄がある場合は、分かりやすい名前を入力しましょう。
     - おすすめの名前：`My-Minutes-Bot`
   - **「Create API key in new project」** をクリックします。
     - これを選ぶと、Googleが自動的に裏側で管理用のプロジェクトを作成してキーを発行してくれます。
     - 「インポートしたプロジェクト」のリストが表示されている場合は、「Default Gemini Project」を選択しても問題ありません。

<!-- 📸 ここにスクリーンショットを挿入：プロジェクト選択のポップアップ画面 -->

6. 数秒の作成処理の後、画面に `AIzaSy` または `AQ.` から始まる長い英数字の文字列（APIキー）が表示されます。

7. キーの右側にある **「Copy」** ボタンをクリックしてコピーし、**メモ帳などに貼り付けて大切に保管**してください。

> **⚠️ 注意**
> APIキーは「パスワード」と同じようなものです。他の人に教えたり、SNSに投稿したりしないように注意してください。

### 6-4. Step 2: Google Apps Script (GAS) プロジェクトの作成

Googleのクラウド上でプログラムを実行するための環境を作成します。

1. **Google Apps Script**（https://script.google.com/）にアクセスします。

<!-- 📸 ここにスクリーンショットを挿入：GASのホーム画面 -->

2. **「新しいプロジェクト」** ボタンをクリックします。

3. 画面左上の「無題のプロジェクト」をクリックし、プロジェクト名を変更します。
   - おすすめの名前：**「ももしたOS用議事録作成」**

<!-- 📸 ここにスクリーンショットを挿入：プロジェクト名の変更画面 -->

4. エディタ画面に最初から書いてある `function myFunction() {}` という文字を**すべて削除**します。

5. 以下のプログラムコードを**すべてコピー**して、エディタに**貼り付け（ペースト）**ます。

```javascript
/**
 * Google Meetの議事録メールを監視し、Gemini APIを使って
 * 指定された「ももしたOS用議事録」テンプレートに自動成形してスレッドに返信するスクリプト
 */

// 設定項目
const SEARCH_QUERY = 'from:gemini-notes@google.com subject:"「" -label:momoshita-os-processed';
const PROCESSED_LABEL = 'momoshita-os-processed';

/**
 * メイン処理：未処理の議事録メールを検索し、フォーマットして返信する
 */
function processMeetMinutes() {
  Logger.log('処理を開始します。');
  
  // 1. 重複防止用のラベルを取得または作成
  let label = GmailApp.getUserLabelByName(PROCESSED_LABEL);
  if (!label) {
    label = GmailApp.createLabel(PROCESSED_LABEL);
    Logger.log('ラベルを作成しました: ' + PROCESSED_LABEL);
  }
  
  // 2. 未処理のメールスレッドを検索
  const threads = GmailApp.search(SEARCH_QUERY);
  Logger.log('未処理のスレッド数: ' + threads.length);
  
  if (threads.length === 0) {
    Logger.log('処理対象のメールはありません。');
    return;
  }
  
  // 各スレッドを処理
  for (let i = 0; i < threads.length; i++) {
    const thread = threads[i];
    const messages = thread.getMessages();
    const latestMessage = messages[messages.length - 1];
    const bodyText = latestMessage.getPlainBody();
    const mailDate = latestMessage.getDate();
    
    Logger.log('スレッドの処理中: ' + thread.getFirstMessageSubject());
    
    try {
      // 3. GoogleドキュメントのURLをメール本文から探す
      let minutesContent = '';
      const docUrlMatch = bodyText.match(/https:\/\/docs\.google\.com\/document\/d\/([a-zA-Z0-9_-]+)/);
      
      if (docUrlMatch && docUrlMatch[1]) {
        const docId = docUrlMatch[1];
        Logger.log('Googleドキュメントを検出しました。ID: ' + docId);
        try {
          const doc = DocumentApp.openById(docId);
          minutesContent = doc.getBody().getText();
          Logger.log('Googleドキュメントからテキストを抽出しました。');
        } catch (e) {
          Logger.log('ドキュメントの読み込みに失敗しました。メール本文を代わりに使用します。エラー: ' + e.toString());
          minutesContent = bodyText;
        }
      } else {
        Logger.log('GoogleドキュメントのURLが見つかりませんでした。メール本文を処理対象にします。');
        minutesContent = bodyText;
      }
      
      // 4. Gemini APIを呼び出して「ももしたOS用議事録」を生成
      const formattedMinutes = generateMomoshitaOSMinutes(minutesContent, mailDate);
      
      // 5. 生成した議事録を自分宛てに送信し、元のスレッド内にぶら下げる（バウンス防止）
      const replyBody = "自動生成された「ももしたOS用議事録」です。

" + formattedMinutes;
      const myEmail = Session.getActiveUser().getEmail();
      GmailApp.sendEmail(myEmail, "Re: " + thread.getFirstMessageSubject(), replyBody, {
        threadId: thread.getId()
      });
      Logger.log('自分宛てにスレッド紐付けメールを送信しました。');
      
      // 6. 処理済みラベルを付与
      thread.addLabel(label);
      Logger.log('処理済みラベルを付与しました。');
      
    } catch (error) {
      Logger.log('エラーが発生しました（スレッドID: ' + thread.getId() + '）: ' + error.toString());
    }
  }
  
  Logger.log('すべての処理が完了しました。');
}

/**
 * Gemini APIを呼び出して、テンプレートに沿った議事録を生成する
 */
function generateMomoshitaOSMinutes(contextText, mailDate) {
```

## 7. 発展編②：NotebookLMで議事録をさらに活用する

**NotebookLM**（https://notebooklm.google.com/）は、Googleが提供するAIリサーチアシスタントです。
Google Meetで作成された議事録をNotebookLMに取り込むことで、**議事録を「記録」から「知識資産」に変える**ことができます。

### NotebookLMとは？

簡単に言うと、「自分専用のAIアシスタント」です。資料（PDFやGoogleドキュメントなど）を読み込ませると、その内容に基づいて質問に回答してくれます。一般的なAIと違い、**読み込ませた資料の内容だけに基づいて回答する**ため、嘘（ハルシネーション）が少なく信頼性が高いのが特徴です。

### 活用フロー

1. **議事録をNotebookLMに取り込む**
   - NotebookLM（https://notebooklm.google.com/）にアクセスし、新しいノートブックを作成します。
   - ソースとして、Googleドライブに保存された**会議のGoogleドキュメント（議事録）**を追加します。
   - 会議で使った配布資料（PDF、スライドなど）も一緒に追加すると、より正確な分析ができます。

2. **AIに質問・分析させる**
   NotebookLMのチャット欄で、以下のような質問や指示ができます：
   - 「この会議の決定事項をまとめて」
   - 「〇〇さんのアクションアイテムを教えて」
   - 「前回と今回の会議で変わった方針はある？」
   - 「次回の会議のアジェンダ案を作成して」

### 活用アイデア

| 活用方法 | 説明 |
|---|---|
| **過去の会議の横断検索** | 複数の議事録をまとめて読み込ませると、「このプロジェクトの議論の流れ」を時系列で把握できます |
| **新メンバーへの引き継ぎ** | 過去の議事録をNotebookLMに入れて、新メンバーが「これまでの経緯を教えて」と質問できるようにします |
| **次回会議の準備** | 前回の議事録を元に「次回のアジェンダ案を作って」と指示すれば、準備時間が短縮できます |
| **音声での振り返り** | NotebookLMの「Audio Overview」機能を使えば、議事録の内容をポッドキャスト形式の音声で聞くことも可能です |

## 8. よくある質問（FAQ）・トラブルシューティング

### Q1. 文字起こし機能が見つかりません
**A.** 会議の主催者が Google Workspace の「Business Standard」以上のプラン、または「Google AI Pro / Ultra」に加入している必要があります。無料のGoogleアカウントでは使えない場合があります。組織の管理者に確認してみてください。

### Q2. 対面会議で使ったら、途中でMeetが勝手に終了してしまいました
**A.** [第4章](#4--知っておきたい設定対面利用時の自動終了を防ぐ方法)の設定を確認してください。Google Meetのホーム画面の設定から「参加者がいない通話を終了」をオフにする必要があります。

### Q3. 会議が終わったのに、議事録メールが届きません
**A.** 以下を確認してください：
- 文字起こし・自動メモ生成が正しく開始されていたか
- 会議中に最低50語程度の発話があったか（短すぎる会議ではメモが生成されないことがあります）
- Gmailの「迷惑メール」フォルダに振り分けられていないか
- 数分〜十数分ほど待ってみてください（即座には届かない場合があります）

### Q4. GASのプログラムを実行したらエラーが出ました
**A.** よくあるエラーと対処法：
- **「GEMINI_API_KEY が設定されていません」** → Step 3のスクリプトプロパティの設定を確認してください。プロパティ名が `GEMINI_API_KEY`（大文字・アンダーバー）と正確に一致しているか確認してください。
- **「Gemini APIエラー (ステータスコード: 400)」** → APIキーが正しくコピーされているか確認してください。前後に余計なスペースが入っていないかもチェックしてください。
- **「ドキュメントの読み込みに失敗しました」** → Googleドキュメントへのアクセス権限がない可能性があります。メール本文を使って処理が続行されるので、結果は確認してみてください。

### Q5. GASのプログラムが5分ごとに動くと、パソコンが重くなりませんか？
**A.** なりません。このプログラムは**Googleのサーバー（クラウド）上で動作**します。ご自身のパソコンの電源が切れていても、ブラウザを閉じていても、Googleが裏側で自動的に実行してくれます。パソコンのメモリやバッテリーに影響はありません。

### Q6. 費用はかかりますか？
**A.** 基本的に無料で利用できます。
- **Google Apps Script**: 無料（1日の実行時間90分の制限がありますが、通常利用では十分です）
- **Gemini API**: 無料枠があり、通常の会議頻度では十分です

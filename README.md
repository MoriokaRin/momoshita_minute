# 桃下村塾 議事録自動作成システム（Momoshita Minutes Bot）

Google Meetの「自動メモ生成（Gemini）」機能と、Google Apps Script (GAS) + Gemini APIを組み合わせ、会議終了後に自動で届く議事録を「ももしたOS用テンプレート」に自動整形し、Gmailスレッド内に配信するシステムです。

## 🚀 使い方（クイックスタート）

サークルメンバー全員がこのシステムを使って会議を行うための、最もシンプルな手順です。難しい初期設定は不要で、すぐに使えます。

### Step 1. 会議の立ち上げと共有
* 会議の主催者（ホスト）が [Google Meet](https://meet.google.com/) で通常通り会議を作成し、参加者にURLを共有して入室します。

### Step 2. 「自動メモ生成（文字起こし）」を開始する
1. 会議が始まったら、画面右下のメニュー（または右上）にある **「自動メモ生成（鉛筆マーク）」** アイコンをクリックします。
2. 表示されたメニューから **「開始」** をクリックします。
   * ※参加者全員に「メモが記録されています」と通知され、AIによる自動記録が始まります。

### Step 3. 普通に会議で話す
* 途中のメモ書きは不要です。議論や対話に集中してください。
   * *(対面会議の場合は、テーブルの中央にPCを1台置いてMeetに入り、マイクで集音するだけでOKです)*

### Step 4. 自動で届く議事録メールを確認する
* 会議が終わってMeetを退出すると、数分後にご自身のGmailへ自動的に以下のメールが届きます。
  1. **Google公式の議事録メール**: 議論の要約と、文字起こしが保存されたGoogleドキュメント（gdocファイル）へのリンクが添付されています。
  2. **ももしたOS用の自動整形議事録**: 同じメールの返信（スレッド）として、桃下村塾のテンプレートに沿って整理された議事録テキストが自動で届きます。

### 💡 裏側で動いているシステム全体の仕組み（全貌）

一般の会議参加者はMeetで話すだけで完了ですが、裏側では本リポジトリのプログラム（GAS）とAIが連携して、以下の**全自動のデータ処理ループ**が動いています。これにより、人間が一切手を加えることなく高品質な議事録が仕上がります。

```text
[ Google Meet 会議終了 ]
       │
       ▼ (数分後)
① Gmailに公式の議事録メールが届く (文字起こしドキュメントのURL入り)
       │
       ▼ (5分ごとにパトロール)
② プログラム (GAS) が自動で新着の議事録メールを検知 
       │
       ▼ 
③ GASが文字起こしテキストを自動で読み込み、Gemini API (AI) に送信
       │
       ▼ (AIがももしたOSテンプレートへ瞬時に要約・整理)
④ 生成された綺麗な議事録を、元のメールスレッドに自動送信 (自分宛て)
```

*   **Google Apps Script (GAS)**: Googleが提供するクラウド上のプログラム実行環境です。ご自身のパソコンがシャットダウンされていても、Googleのサーバー上で24時間365日パトロールを実行し続けます。
*   **Gemini API (AI)**: Googleの誇る最新AIモデルです。生の文字起こしデータから「議題」「決定事項」「誰が何をいつまでにやるか（アクションアイテム）」を自動判別し、完璧なビジネス文書に要約します。

## 🛠️ システムの初期設定手順（管理者・設定担当者向け）

このシステムをご自身のGoogle Workspace環境に新規で導入・設定したい管理者向けのセットアップ方法です。

詳しい設定の図解や手順は、**[MomoshitaOS_Minutes_Setup_Manual.md](MomoshitaOS_Minutes_Setup_Manual.md)** または **[Minutes_HandsOn.md](Minutes_HandsOn.md)** をご覧ください。

1. **Gemini APIキーの取得**: [Google AI Studio](https://aistudio.google.com/) にて、無料枠で動作するAPIキーを取得します。
2. **GASの作成**: [Google Apps Script](https://script.google.com/) にて、`meeting_formatter.js` のプログラムコードを貼り付け、APIキーを書き込みます。
3. **自動実行トリガーの登録**: GASの「トリガー」設定から、プログラムを「5分ごと」に実行するタイマーをセットします。

## 💡 本システムの特長

*   **完全全自動稼働**: Googleのクラウド上で動作するため、パソコンの電源が切れていても自動でGmailを監視して議事録を作ります。
*   **バウンス（不達）エラー防止機能**: Google Meetの送信元アドレス（`gemini-notes@google.com`）は送信専用のため返信するとエラーになります。本システムは、**「自分宛てに送信しつつ元のスレッドに紐付ける」** ことで、エラーを100%防ぎながらスレッド配信を実現しています。
*   **最新AIモデル対応**: クォータ（利用制限）エラーを回避するため、無料枠が安定して動作する最新の **`gemini-2.5-flash`** モデルを標準採用しています。

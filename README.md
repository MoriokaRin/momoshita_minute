# 桃下村塾 議事録自動作成システム（Momoshita Minutes Bot）

Google Meetの「自動メモ生成（Gemini）」機能と、Google Apps Script (GAS) + Gemini APIを組み合わせ、会議終了後に自動で届く議事録を「ももしたOS用テンプレート」に自動整形し、Gmailスレッド内に配信するシステムです。

## 🛠️ 導入の流れ（クイックスタート）

本システムを導入するための大まかなステップです。詳細な設定手順や画面付きの解説は、[Minutes_HandsOn.md](Minutes_HandsOn.md) を参照してください。

### Step 1. Gemini APIキーの取得
1. [Google AI Studio](https://aistudio.google.com/) にサインインします。
2. 左メニューの **「Get API key」** から、**「Default Gemini project」**（または新規プロジェクト）でAPIキーを発行し、コピーします。
   * ※通常の `@gmail.com` アカウントであれば、1日1500回まで完全に無料で利用できます。

### Step 2. GASプロジェクトの作成とコードの貼り付け
1. [Google Apps Script](https://script.google.com/) にアクセスし、新規プロジェクトを作成します。
2. `meeting_formatter.js` のコードをすべてコピーし、GASエディタに貼り付けます。
3. コード内の以下の部分（91行目付近）をご自身が取得したAPIキーに書き換えます。
   ```javascript
   const apiKey = 'ここに取得したAPIキーを貼り付ける';
   ```
4. プロジェクトを保存（`Ctrl + S` または `Cmd + S`）します。

### Step 3. トリガーの登録
1. GASエディタの左メニューにある **⏰ トリガー（目覚まし時計マーク）** をクリックします。
2. 右下の「トリガーを追加」をクリックし、以下のように設定して保存します。
   * **実行する関数**: `processMeetMinutes`
   * **イベントのソース**: `時間主導型`
   * **トリガーのタイプ**: `分ベースのタイマー`
   * **時間の間隔**: `5分ごと`

## 💡 本システムの特長とアップデート内容

*   **完全全自動稼働**: Googleのクラウド上で動作するため、パソコンの電源が切れていても自動でGmailを監視して議事録を作ります。
*   **バウンス（不達）エラー防止機能**: Google Meetの送信元アドレス（`gemini-notes@google.com`）は送信専用のため返信するとエラーになります。本システムは、**「自分宛てに送信しつつ元のスレッドに紐付ける」** ことで、エラーを100%防ぎながらスレッド配信を実現しています。
*   **最新AIモデル対応**: クォータ（利用制限）エラーを回避するため、無料枠が安定して動作する最新の **`gemini-2.5-flash`** モデルを標準採用しています。
